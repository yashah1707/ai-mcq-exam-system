# 🎓 Viva Voce & Evaluation Guide
## AI-Based Student Performance Monitoring & Adaptive Test Platform

### 1. Core Concept
**Q: How is this system "Adaptive"?**
A: The system uses a **Rule-Based Adaptive Engine** (Node.js) that runs after every test attempt.
- It analyzes your **Weak Topics** (topics with <50% accuracy).
- It checks **Time Spent** (identifying guessing vs. struggling).
- The next test is generated by:
  1.  Fetching 60% questions from your **Weak Topics** (to improve them).
  2.  Fetching 20% from **Strong Topics** (for revision).
  3.  Fetching 20% **New Topics** (for exploration).
- Difficulty is adjusted: If you score >80%, next test has more "Hard" questions.

### 2. The Role of AI
**Q: Where is the AI/ML used?**
A: We use a separate **Python Microservice (Flask + Scikit-Learn)** for advanced analytics, NOT for simple CRUD.
- **Clustering**: We use logic to classify students into profiles like "Cautious Expert", "Rusher", or "High Performer".
- **Trend Analysis**: Utilizing Linear Regression (`sklearn`) to calculate the slope of your performance graph (Improving / Declining).
- **Readiness Score**: A weighted algorithm considering Accuracy, consistency (Std Dev), and Topic Coverage to predict "Job Readiness".

### 3. Architecture
**Q: Why a microservice?**
A: To separate concerns. Node.js is great for I/O and handling thousands of requests (taking exams). Python is superior for Data Science and Math operations. This decoupled architecture allows us to scale the AI engine independently.

### 4. Database Design
**Q: How do you track progress over time?**
A: We use a `PerformanceAnalytics` collection that tracks cumulative accuracy per topic. We also store `ExamAttempt` logs to replay history and calculate trends.

### 5. Future Scope
- Add **Collaborative Filtering** to recommend tests based on similar students.
- Implement **Deep Learning (LSTM)** to predict exact marks in the final placement exam.
